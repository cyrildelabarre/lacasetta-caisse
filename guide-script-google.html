🍕 La Casetta · Script Google (Apps Script)

Guide du script Google

Le « cerveau » côté cloud : il reçoit les ventes, remplit le Google Sheet, calcule les KPI et envoie les e-mails. Ce guide explique comment l'utiliser et à quoi sert chaque fonction.

📂 Ouvrir le script

  1. Va sur script.google.com et ouvre le projet La Casetta (celui lié à ton URL de caisse).
  2. L'onglet < > Éditeur (à gauche) affiche le code.

▶️ Exécuter une fonction

  1. En haut, ouvre le menu déroulant des fonctions (au milieu de la barre d'outils).
  2. Choisis la fonction à lancer (ex. rebuildAll).
  3. Clique ▶ Exécuter.
  4. La 1ʳᵉ fois, Google demande une autorisation → « Examiner les autorisations » → choisis ton compte → « Autoriser ».
  5. Le Journal d'exécution (en bas) affiche le résultat.

🚀 Redéployer (après une mise à jour du code)

  1. Colle le nouveau code → Enregistrer (Ctrl/Cmd + S).
  2. Déployer ▸ Gérer les déploiements.
  3. Clique le crayon ✏️, choisis Version : Nouvelle version, puis Déployer.

Important : « Nouvelle version » est l'étape clé — sans elle, l'URL sert l'ancien code. L'URL /exec, elle, ne change pas. Exécuter une fonction depuis l'éditeur utilise le code enregistré (pas besoin de redéployer) ; redéployer ne sert que pour l'URL appelée par la caisse.

À faire une fois Mise en place des automatismes
createWeeklyTrigger

Programme l'e-mail récap hebdomadaire.

Quand : une fois. Envoi chaque vendredi ~22h.

createDailyEmailTrigger

Programme l'e-mail récap du jour.

Quand : une fois. Envoi chaque soir ~22h15.

createOpenTrigger

Recalcule tous les onglets à chaque ouverture du Google Sheet.

Quand : une fois.

setupFacebook

Enregistre ton ID de page + jeton Facebook (dans les réglages privés).

Quand : une fois, après avoir collé tes identifiants dans la fonction. Facultatif (Facebook).

createDailyFacebookTrigger

Programme la publication Facebook de l'emplacement du jour.

Quand : une fois, après setupFacebook. Publie chaque jour ~10h.

Au besoin Maintenance
rebuildAll

Recalcule tous les onglets de synthèse à partir de l'onglet Transactions.

Quand : après une mise à jour du code, ou si un onglet semble faux.

deleteLastSale

Supprime la dernière vente enregistrée, puis recalcule les onglets.

Quand : pour retirer un ticket de test ou une erreur récente.

Test Envoi / vérification immédiate
sendDailyReport

Envoie tout de suite l'e-mail récap du jour.

Quand : pour tester l'e-mail quotidien.

sendWeeklyReport

Envoie tout de suite l'e-mail récap de la semaine.

Quand : pour tester l'e-mail hebdo.

testPostFacebook

Publie un post Facebook de test [TEST] sur ta page.

Quand : pour vérifier la connexion Facebook. Supprime le post ensuite.

Automatique Ne pas exécuter à la main
doPost

Reçoit les ventes envoyées par la caisse, les écrit dans Transactions et recalcule les onglets.

Déclenché automatiquement à chaque vente.

doGet

Répond aux demandes de la caisse : charger l'historique, annuler une vente, supprimer la dernière, recalculer.

Déclenché automatiquement par le POS.

cancelTicket

Passe un ticket au statut « Annulé » et recalcule. Appelée par la caisse lors d'une annulation.

Ne pas lancer seule (a besoin d'un identifiant).

postTodayLocation

Publie l'emplacement du jour sur Facebook.

Déclenchée par le programmateur Facebook.
